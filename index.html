Your Essential Git Commands Shortlist

1. Check current status:

git status

Use Case: Before adding or committing, this command shows you which files have been modified, are staged for commit, or are untracked. Essential for understanding your repository's state.

2. Stage all changes:

git add .

Use Case: Adds all new, modified, and deleted files in your current directory and its subdirectories to the staging area, preparing them for the next commit.

3. Create a new file (Windows PowerShell):

New-Item YourFileName.txt -ItemType File

Use Case: Quickly create new files directly from your VS Code terminal (if using PowerShell) for documentation, configuration, or new code files.

3. Create a new file (Bash/Mac/Linux):

touch YourFileName.txt

Use Case: The equivalent of `New-Item` for Bash/Unix-like environments, commonly used to create empty files.

4. Commit with a descriptive message:

git commit -m "Your descriptive commit message here"

Use Case: Saves your staged changes to your local repository's history. The message should clearly describe the changes made in that commit.

Refactor Commit Scenarios:

  • feat: Add user authentication module (New Feature)
  • fix: Resolve critical bug in login flow (Bug Fix)
  • docs: Update README with setup instructions (Documentation)
  • refactor: Improve naming conventions in user service (Code Refactoring)
  • chore: Update dependencies (Maintenance/Chores)

5. Initial push (only used initially for a new branch):

git push -u origin main

Use Case: This command is typically used for the *first* push of a new branch (like `main`) to a remote repository. The `-u` flag sets the upstream branch, meaning `git push` (without arguments) will work for subsequent pushes.

6. Subsequent pushes:

git push

Use Case: After the initial push, this command is used to upload all your new local commits to the remote repository. Simpler because the upstream is already set.

Juicy Extra 1: Interrupt a running command:

Ctrl + C

Use Case: Crucial for stopping long-running processes, commands that are stuck, or if you accidentally initiated a command you didn't intend to. Works in most terminal environments (Windows, macOS, Linux).

Juicy Extra 2: Print Working Directory:

pwd (Mac/Linux/Bash)
Get-Location or cd (Windows PowerShell, `cd` without arguments shows current directory)

Use Case: Always know where you are! This command displays the full path of your current directory in the terminal. Essential for navigation and ensuring you're running commands in the correct location.

  1. Create main folder: project-warehouse
  2. Open in VS Code (Right-click → Open with Code)
  3. Create project subfolders with meaningful names
  4. Move keeper projects into organized structure
  5. Delete unwanted snippets and experiments
  6. Add README.md to document your collection
  7. Initialize Git and push to GitHub

Start with 1-2 projects, then add incrementally. This keeps commit history clean and makes each addition meaningful.

🤖File Organization Script

Python script that automatically organizes files by type:

  1. Install Python 3.x on your system
  2. Create new file: organize_files.py
  3. Import required modules: import os, shutil, pathlib
  4. Set watched directory: WATCHED_DIR = Path.home() / "Downloads"
  5. Set destination: DEST_DIR = Path.home() / "Organized"
  6. Define file type mappings for Images, Documents, Code, etc.
  7. Create organize_files() function
  8. Loop through files in watched directory
  9. Move files to appropriate category folders
  10. Run script: python organize_files.py
Save as organize_files.py and run with python organize_files.py

⏰Automation Schedule

Windows Task Scheduler:

  1. Open Task Scheduler
  2. Create Basic Task
  3. Set trigger (hourly/daily)
  4. Action: Start Program
  5. Point to your .exe file

macOS/Linux Cron Setup:

  1. Open terminal
  2. Edit crontab: crontab -e
  3. Add hourly job: 0 * * * * /usr/bin/python3 /path/to/organize_files.py
  4. Add startup job: @reboot /usr/bin/python3 /path/to/organize_files.py
  5. Save and exit editor
  6. Verify with: crontab -l

🚀Quick Setup Scripts

Project Creator Script Setup:

  7. Initialize README: echo "# $PROJECT_NAME" > README.md
  8. Stage files: git add .

  10. Make executable: chmod +x create_project.sh
  11. Use script: ./create_project.sh my-new-app

💎Organization Pro Tips

  • Naming Convention: Use kebab-case for folders (my-project)
  • Version Control: Commit early, commit often with clear messages
  • Documentation: Every project needs a README, even small ones
  • Dependencies: Use package.json even for vanilla JS projects
  • Backup Strategy: GitHub + local backups = peace of mind
  • Clean Commits: One feature per commit, descriptive messages

🚨Common Mistakes to Avoid

  • Don't: Commit node_modules or build files
  • Don't: Use spaces in folder names (use-hyphens-instead)
  • Don't: Force push to main branch
  • Don't: Commit without testing first
  • Don't: Use generic commit messages like "update"
  • Don't: Store sensitive data in public repos
When in doubt, create a branch. Branches are cheap, broken main branches are expensive.
